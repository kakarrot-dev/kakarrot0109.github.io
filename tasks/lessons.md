## 2026-07-15

### 博客文章应延续现有序号与调性

- 现象：把新文章设计成了不带序号的专题文章。
- 根因：只依据内容规模选择文章形式，没有先核对 `src/content/writing/` 的既有标题结构。
- 解决：本次文章使用下一个序号 `No. 20`，写入 `src/content/writing/`，延续个人叙事开场和编号主体结构。
- 以后注意：设计新文章前先检查现有最大序号与近期文章调性；除非用户明确要求专题形式，否则沿用既有序号。


### 规划文档不能放在构建输出目录

- 现象：运行 `npm run build` 后，`docs/superpowers/` 下未提交的 spec 和 plan 被 Astro 清空。
- 根因：`docs/` 同时被定义为 GitHub Pages 构建输出目录和规划文档目录，职责冲突。
- 解决：将规划文档迁到 `tasks/specs/` 和 `tasks/plans/`，`docs/` 只保留构建产物。
- 以后注意：持久源文件不能放在会被构建工具清空的输出目录中。

### 会阻断部署的基线错误应先处理

- 现象：文章任务开始前发现 `npm run check` 存在既有错误，最初准备记录后继续。
- 根因：只考虑错误是否由本次文章引起，没有继续核对部署工作流的执行顺序。
- 解决：确认 GitHub Actions 在构建前强制运行类型检查，因此先修复基线错误，再编写文章。
- 以后注意：基线检查失败时同时查看部署工作流；只要错误会阻断交付，即使与当前内容改动无关，也应先说明影响并确认处理顺序。

## 2026-07-16

### GitHub 账号迁移要检查仓库内全部手写链接

- 现象：网站链接已更新为新账号，但项目仓库 README 的徽章和作者链接仍指向旧账号。
- 根因：账号迁移时只更新了网站入口，没有检索关联仓库中的手写链接。
- 解决：同时检查网站源码、构建页面、README、徽章目标和作者署名链接。
- 以后注意：GitHub 账号或仓库路径变化后，统一检索旧账号，并验证 GitHub 页面与 Raw 资源均可访问。

## 2026-07-20

### 资料型文章不能为了叙事性删掉核心查阅价值

- 现象：把完整对照资料改写成个人实践文章后，只剩一张职责表和一个目录代码块，读者难以快速复制入口配置或核对两端加载差异。
- 根因：结构收敛时只保留了叙事主线，把“避免手册式堆砌”误解成“删除核心对照和示例”。
- 解决：保留个人实践主线，同时恢复两张精简表和三个经过事实核验的核心代码块；删除未核验、版本敏感且无助于减少重复维护的细节。
- 以后注意：改写资料型文章时，先定义最小查阅集；叙事负责解释为什么，表格和示例负责让读者快速核对与复用。
