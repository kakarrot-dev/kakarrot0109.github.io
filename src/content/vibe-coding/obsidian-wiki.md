用 Obsidian + AI Agent 协作维护的本地知识库。核心原则五条：原文可追溯，加工有结构，标签可检索，附件可归档，规则可执行。

### 目录架构

| 目录 | 用途 |
|---|---|
| `00-网页收藏/` | Web Clipper 采集原文，保留原貌不改 |
| `01-产品管理/` | 市场、竞品、行业、需求、产品机会和风险 |
| `02-项目管理/` | 项目目标、计划、决策、进度和复盘 |
| `03-知识提炼/` | 可复用的概念、原则、方法和长期知识 |
| `04-兴趣爱好/` | 兴趣、生活、个人探索和非工作主题 |
| `99-附件/` | 图片、PDF、音视频等附件 |
| `templates/` | Obsidian 笔记模板（采集/市场/项目/知识） |

### 笔记类型

**采集类笔记** — 放在 `00-网页收藏/`，保留原文主体不润色不改，来源 URL 必须保留。需要提炼时新建加工笔记，不在原文上直接改写。

**加工类笔记** — 从采集内容迁移过来时重新组织结构，不复制整篇原文。结论必须能追溯到来源、证据或明确的推理。

**知识提炼类笔记** — 只放长期可复用的概念和方法，用 `source` 或脚注保留证据链。

### 协作规范

- Agent 操作规则写在 `AGENTS.md`，Claude Code 入口在 `CLAUDE.md`
- 非琐碎任务先写 spec → 确认后再写 plan → 确认后再执行
- 被纠正后的经验写入 `tasks/lessons.md`
- 默认不自动 commit 或 push
