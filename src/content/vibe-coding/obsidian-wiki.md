一个面向 Obsidian 和 AI Agent 协作的本地知识库。

## 为什么做

传统的文件夹+标签知识管理，AI Agent 看不懂。我要一个既能自己用、又能让 Agent 自动写入和检索的知识库。

## 核心理念

**原文可追溯，加工有结构，标签可检索，附件可归档，规则可执行。**

知识库分成五个区域：

- **00-网页收藏** — 采集原文，不润色不重排，只保留来源
- **01-产品管理** — 市场、竞品、行业、需求研究，结论必须能追溯到来源
- **02-项目管理** — 目标、范围、决策、进度和复盘
- **03-知识提炼** — 长期可复用的概念、方法和规则，重新组织不复制原文
- **04-兴趣爱好** — 非工作主题资料

## 与 AI Agent 协作

这是这个项目的核心创新。整个知识库的目录结构、Properties 定义、标签体系和编写规范都经过设计，让 AI Agent 能理解上下文、识别笔记类型、正确放置和检索内容。

Agent 操作规则写在 `AGENTS.md` 里，非琐碎任务走 spec → plan → 开发的标准流程。被纠正的经验写到 `tasks/lessons.md`，防止重复犯错。

## 当前状态

知识库在持续增长中，Agent 协作流程已经跑通，Obsidian 插件生态兼容良好。
