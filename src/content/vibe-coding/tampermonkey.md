这是我的个人 Tampermonkey / Violentmonkey 用户脚本仓库。它从一个很具体的问题开始：**当我每天反复使用一个网页时，与其持续忍受同样的体验摩擦，不如为它补上一层符合自己习惯的界面。**

## 为什么做

我在 X 上主要进行内容阅读，但默认页面把大量空间交给右侧栏、浮动按钮和偏窄的主内容区。信息仍然能读，却需要在无关元素之间不断重新聚焦；媒体尺寸和行间距也不适合长时间浏览。

这些问题稳定、重复，而且不需要改变平台本身的业务逻辑。它们很适合用一段小型用户脚本解决。

## 关键产品决策

- **先解决自己的高频问题** — 不预设一个庞大的脚本平台，从真实使用频率最高的 X 阅读场景开始
- **只调整表现层** — 当前脚本只使用 CSS 和 `GM_addStyle`，不拦截请求，也不接管发帖、互动等业务行为
- **保持低权限和易恢复** — 脚本随时可以关闭，页面原有能力不被修改
- **兼容实际访问入口** — 同时匹配 `x.com` 和 `twitter.com`，避免域名切换导致脚本失效

## 当前脚本：X 阅读增强

脚本隐藏右侧栏和右下角浮动按钮，加宽主内容区，调整页面间距、正文行高和媒体高度，同时保留左侧导航。目标不是重新设计 X，而是减少与阅读无关的视觉噪声，让时间线、个人主页和帖子详情拥有更连续的浏览节奏。

## 如何验证体验

这类脚本的正确性不能只看代码是否执行。我会在登录状态下使用 1280、1440 和 1920 三种常见宽度，分别检查首页、个人主页、帖子详情和站内跳转：主栏需要明显更宽，右侧空白减少，浮钮消失，同时左侧导航和核心交互保持可用。

## 当前状态

仓库目前从一个 X 阅读增强脚本起步。后续是否增加脚本，取决于真实使用中是否出现足够稳定、足够高频的问题；它不会为了数量被包装成通用脚本平台。
