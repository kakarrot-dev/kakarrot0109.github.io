# 写作指南

基于 yanliudesign.live 文章风格分析提炼。

---

## 1. 核心风格定位

**个人叙事 + 结构化经验分享**，介于博客笔记与轻度分析之间。

读者感知：亲切、真诚、有洞察。像朋友分享经验，而非专家说教。

## 2. 语气

| 维度 | 要求 |
|------|------|
| **基调** | 温暖、鼓舞人心、有反思性 |
| **正式度** | 中等偏口语化，使用缩略形式（it's, I've, don't） |
| **情感** | 积极为主，允许暴露脆弱（迷茫、挫折）→ 最终导向成长/反思 |
| **第一人称** | 强烈使用 "I"，全文贯穿。结尾转为 "You" 直接对读者说话 |
| **态度** | 真实不设防，不堆砌术语，不教条 |

**语气禁区：**

- ❌ 冷眼旁观式客观分析
- ❌ 学术论文腔
- ❌ 过度营销/推销感
- ❌ 居高临下的说教

## 3. 标题格式

```
No. {数字} {冒号或句点} {主标题}
```

示例:
- No 44: Vibe Coding Workflow for Non-Programmers
- No 42: Three Years in a Row Winning Hackathons — Witnessing Growth!
- No 7: What I Learned from My Spotify Layoff a Year Ago

规则:
- 英文标题，首字母大写（Title Case）
- 标题包含一个钩子（具体数字、事件、引人好奇的角度）
- 可带副标题，放在文章开头紧随标题之后

## 4. 文章结构

```
[标题]
[副标题/设问] — 可选，用于点明核心论点

1. 开场钩子
   - 个人经历/事件切入（"Last week, I..." / "Looking back..."）
   - 交代背景（时间、地点、场景）
   - 点出核心问题或矛盾

2. 主体（结构化要点）
   - 2-6 个编号/加粗要点
   - 每点一个小标题 + 2-5 段正文
   - 段内用 bullet points 展开子项（每点不超过 3-4 个子项）
   - 每段 2-5 句，一个观点一段

3. 升华/总结
   - 提炼核心观点（1-2 段）
   - 一句"金句"收尾（加粗）
   - 可加 ❤️ 或号召性结尾
```

## 5. 段落与句子

- **段落长度**: 2-5 句，控制在手机阅读舒适长度
- **句子长度**: 10-22 词为主，偶用 5-7 词短句做节奏变化，极少 30 词以上长句
- **过渡词**: But, Yet, However, Then, Honestly, After that — 自然口语化衔接
- **视觉效果**: 大量空行断句，视觉上"呼吸感"强

## 6. 用词特征

- **动作动词**: building, creating, shifting, rewriting, connecting
- **情感词汇**: thrilling, rewarding, terrifying, buzzing, bittersweet
- **比喻语言**: dots, waves, crossroads, curveball, magnet, spark
- **术语**: 适度使用行业术语，首次出现需上下文自解释
- **缩略形式**: 持续使用（it's, I've, didn't, there's, don't）

## 7. 格式要求

| 元素 | 使用规则 |
|------|----------|
| **加粗** | 关键概念、公司/产品名称、核心金句加粗 |
| **斜体** | 极少使用，倾向加粗替代 |
| **引用块** | 不使用传统 blockquote |
| **无序列表** | 要点内部展开子项时使用（每点 ≤ 4 项） |
| **编号列表** | 主体结构用 1️⃣2️⃣3️⃣ emoji 编号 |
| **图片** | 每节之间穿插 1 张（场景照/产品图/示意图） |
| **分隔线** | 极少使用，倾向空行分隔 |
| **Emoji** | 适度点缀（2-5 个/篇），用于增强情感而不替代文字 |

## 8. 数字编号规则

主体要点使用 emoji 编号：

```
1️⃣ First Point Title
2️⃣ Second Point Title
3️⃣ Third Point Title
```

原因：视觉上更突出，适合社交传播。

## 9. 英文 vs 中文

文章正文使用**英文**撰写（面向国际读者）。

中文使用场景:
- 本文档（写作指南）
- 项目 README
- 开发文档/注释

## 10. 文章长度

| 类型 | 长度 | 适用场景 |
|------|------|----------|
| 短文 | 500-800 词 | 项目展示、快速分享 |
| 中篇 | 1000-2000 词 | 经验总结、方法分享 |
| 长文 | 2000-3500 词 | 深度分析、系列文章 |

## 11. 标签规则

文末使用标签/话题标记，3-6 个，覆盖:

1. 技术主题（如 AI, Product Design, Design）
2. 文章类型（如 Career, Personal Growth, Strategy）
3. 工具/公司（如 Gemini, Spotify, Notion）

## 12. 检查清单

写完后逐项检查:

- [ ] 标题格式正确（No. XX: 标题）
- [ ] 开场有个人钩子
- [ ] 结构有 2-6 个编号要点
- [ ] 段落控制在 2-5 句
- [ ] 句子以短中句为主
- [ ] 加粗了关键概念和金句
- [ ] 结尾有升华和号召
- [ ] Emoji 使用克制（2-5 个）
- [ ] 读起来像人在说话，不是机器
